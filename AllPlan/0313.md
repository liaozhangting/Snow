第一阶段：环境初始化（09:00 - 11:00）

执行指令： 安装最基础的运行环境。



更新系统： sudo apt update \&\& sudo apt install -y openjdk-11-jdk mosquitto mosquitto-clients



注意： mosquitto 是 C 写的，裸机运行只需不到 10MB 内存，完美。



安装 Kafka： 直接去官网下载二进制包（tgz），解压到 /opt/kafka。



安装 Go 环境： 如果没有，直接 sudo apt install golang-go。



第二阶段：核心组件“内存压榨”启动（11:00 - 13:00）

执行指令： 必须手动限制 JVM，否则 4G 内存瞬间爆表。



启动 Zookeeper： 修改 zookeeper.properties，限制最大连接数。



启动 Kafka：



关键： 启动前执行 export KAFKA\_HEAP\_OPTS="-Xmx512M -Xms512M"。



命令：bin/kafka-server-start.sh -daemon config/server.properties。



检查内存： 执行 free -h，确保此时已用内存不超过 1.5G。



第三阶段：代码逻辑适配（14:00 - 18:00）

执行指令： 因为没有 Docker 网桥，所有通信直接走 localhost 或公网 IP。



gRPC 路径： 5090 服务器开启 50051 端口。



异步日志： Go 程序直接连接 localhost:9092 (Kafka) 和 localhost:1883 (MQTT)。



配置硬编码： 所有的 IP:Port 直接写在代码的 const 里，不要再折腾 Nacos 动态读取。



第四阶段：云端模拟与测试（19:00 - 22:00）

执行指令： 验证闭环。



日志流水： 让雪毛儿在控制台开一个 kafka-console-consumer.sh，看着你的采摘日志一条条跳出来。



指令下发： 通过 mosquitto\_pub 命令手动模拟云端下发模型更新指令。



稳定性观测： 运行 top -p \[Kafka进程PID]，观察在高频率日志下 CPU 和内存的波动。
