\### 2. 编写 Kafka 生产者 Demo (耗时预估：40min)



让雪毛儿在端侧程序中集成 Kafka 客户端（推荐使用 `segmentio/kafka-go`，纯 Go 实现，不依赖 CGO，更省内存）。



\*\*执行逻辑：\*\*



1\. 启动一个 Goroutine。

2\. 模拟每 2 秒产生一条 JSON 日志：`{"robot\_id": "SN-001", "action": "picking", "status": "success", "ts": 1710316800}`。

3\. 将日志写入 `robot\_logs` 话题。

\### 3. 编写 MQTT 指令监听器 (耗时预估：40min)



在边缘侧程序（5090 服务器逻辑）中集成 MQTT 客户端（推荐 `eclipse/paho.mqtt.golang`）。



\*\*执行逻辑：\*\*



1\. 订阅 `model\_update` 话题。

2\. 编写回调函数：收到消息后解析 JSON，打印一行醒目的日志：`\[SYSTEM] Received remote update command: Version 2.0. Downloading...`。

3\. 模拟下载行为：`time.Sleep(time.Second \* 3)`。

\### 5. 第二阶段验收标准



1\. \*\*链路 A：\*\* 端侧程序运行后，云端 Kafka 消费者能实时刷出 JSON 日志。

2\. \*\*链路 B：\*\* 手动发布 MQTT 消息后，边缘侧程序能立刻做出响应并触发“模拟下载”逻辑。

3\. \*\*资源监控：\*\* 运行 `top` 命令，观察 Go 进程的 CPU 占用。如果超过 5%，说明代码里有死循环，需要 review 他的 `for` 循环。
